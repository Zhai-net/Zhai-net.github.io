# 代码规范与维护性审查结果

## 已解决的问题

1. **重复 HTML 结构**：多个页面原先分别维护页头、页脚、主题脚本、灯箱和返回顶部按钮，容易出现链接、文案和属性不一致。现在统一由模板和站点配置生成。
2. **站点配置分散**：邮箱、外部链接、更新时间、统计脚本地址散落在 HTML 和数据文件中。现在集中在 `site.config.mjs`。
3. **单个 JavaScript 文件职责过多**：原 `common.js` 同时处理导航、主题、动画、灯箱、成员卡片、论文搜索等。现在按功能拆分源码，再构建为单文件。
4. **渲染逻辑耦合**：原 `render.js` 集中处理全部数据类型。现在按人员、论文、新闻、图库和站点元信息拆分，并通过统一入口派发内容渲染完成事件。
5. **CSS 长期追加覆盖**：原样式表接近 2000 行且带有多个版本补丁段。现在按基础、首页、子页面、论文、交互、人员、性能和无障碍分层维护，发布时仍合并为一个文件。
6. **死代码**：删除了没有对应界面的旧论文筛选逻辑和未使用的 `navigation.json`。
7. **脆弱的脚本执行顺序**：动态内容交互现在使用事件委托，并监听 `zhai:content-rendered`，降低渲染脚本与交互脚本之间的隐式依赖。
8. **重复初始化风险**：动态组件通过 `data-*` 状态标记，避免重复绑定事件或重复插入按钮。
9. **数据缺少校验**：新增成员 ID、分类、邮箱、图片路径、论文必填字段、年份、重复 DOI、图库资源和新闻字段检查。
10. **生成文件易被误改**：生成文件顶部增加来源说明，并通过 `npm run build:check` 检测手工修改或过期产物。

## 维护入口

- 站点配置：`site.config.mjs`
- 页面正文：`src/pages/`
- 样式源码：`src/css/`
- 通用交互：`src/js/common/`
- 数据渲染：`src/js/render/`
- 内容数据：`assets/data/people.js`、`publications.js`、`news.js`、`gallery.js`
- 构建：`npm run build`
- 全面检查：`npm run check`
